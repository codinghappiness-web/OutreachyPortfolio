Featured Contribution

RamaLama: learn how RamaLama makes working with AI boring

Outreachy Project • Main Task Assigned
View Issue View Repository

Evaluation of RamaLama across multiple model transports (Ollama, Hugging Face, ModelScope, OCI) to assess suitability for RAG-based Fedora packaging workflows.

I set up and tested RamaLama to understand how it makes working with AI models easier using RAG and different model transports.

What I Did

  • Installed RamaLama on my system
  • Pulled and ran models using different sources (HuggingFace, Ollama, OCI)
  • Ran prompts like: “What are the Four Foundations of the Fedora Project?”
  • Documented commands, outputs, and results

Challenges I Faced

  • The Granite 7B model could not run due to limited memory
  • I experienced container timeout issues
  • I ran into GitHub Container Registry (GHCR) authentication errors

How I Solved Them

  • Switched to smaller models like Qwen, TinyLlama, and Phi-3
  • Used the --nocontainer option to avoid Docker issues
  • Fixed Docker and authentication errors through debugging

What I Learned

  • Different model sources (HuggingFace, Ollama, OCI) work differently
  • Running AI locally depends a lot on system resources
  • Debugging and adapting are very important when working with AI tools

My Insight: Does RamaLama Make AI Boring?

While working on this project, I started wondering if using tools like RamaLama can sometimes feel boring or repetitive.

I tested different models from HuggingFace, Ollama, and ModelScope using the same question:

“What are the Four Foundations of the Fedora Project?”

Most of the models did not give the exact or complete answer. Some answers were partly correct, while others were unclear.

Because of this, I had to repeat the same prompt across multiple models, and it started to feel repetitive and a bit frustrating.

However, this also taught me something important. Even though RamaLama makes it easy to run models, the quality of the answers depends on the model itself, how it was trained, and how it understands prompts.

So while RamaLama makes the process easier, the results are not always perfect or predictable.

In a way, the experience was both:

  • Boring, because I had to repeat the same task many times
  • Insightful, because it showed the real limitations of AI models

It helped me understand that:

  • Running AI models is becoming easy
  • But getting accurate and reliable answers is still a challenge
